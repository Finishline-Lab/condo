Supported Operating Systems

Arguments

The following optional arguments are available within bower.

NameTypeDefaultRequiredDescription
run_argsstringnullYesThe arguments for the run command.
run_optionsstring${env:RUN_OPTIONS}NoAdditional options to use when executing the run command.
run_pathstring${global:working_path}NoThe path in which to execute run.
run_waitbooleantrueNoA value indicating whether or not to wait for exit.
run_quietboolean${global:quiet}NoA value indicating whether or not to avoid printing output.
run_securebooleanfalseNoA value indicating whether or not to avoid printing secure information for public builds.

Global Arguments

The following global arguments are used by bower:

NameTypeDefaultDescription
base_pathstring$PWDThe base path in which condo was executed.
working_pathstring${global:base_path}The working path in which condo should execute shell commands.
quietbooleanfalseA value indicating whether or not to suppress output when executing condo.

Examples

Condo can execute any managed executable, regardless of the platform. On OS X and Linux, the arguments are passed to the mono bootstrapper. We hope to remove this dependency in the future, but at the present time, condo itself relies on mono, so this dependency would always be available.
