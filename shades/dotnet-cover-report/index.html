dotnet-cover-report

Generates a human-readable report for code coverage results.

Supported Operating Systems

Arguments

The dotnet-cover-report shade accepts the following arguments:

NameTypeDefaultRequiredDescription
targetstring${global:target_test_path}/*-coverage.xmlNoThe path containing the code coverage results that should be parsed.
output_pathstring${global:target_test_path}NoThe path where the report will be emitted.
typestringnullYesThe type of the reports to generate.
src_pathstringnullNoThe path(s) containing source files for reference.
history_pathstringnullNoThe path where historical coverage reports should be retained.
assembly_filterstringnullNoThe filter(s) used to remove specific assemblies from the report.
class_filterstringnullNoThe filter(s) used to remove specific classes from the report.
waitbooleantrueNoA value indicating whether or not to wait for exit.
quietboolean${global:quiet}NoA value indicating whether or not to suppress standard output when generate code coverage report.
All arguments are prefixed by dotnet_cover_report_.

Global Arguments

The following global arguments are used by dotnet-cover-report:

NameTypeDefaultDescription
base_pathstring$PWDThe base path in which Condo was executed.
working_pathstring${global:base_path}The working path in which Condo should execute shell commands.
target_pathstring${env:BUILD_BINARIESDIRECTORY} -OR- ${global:base_path}/artifactsThe path where build artifacts and results should be stored.
target_test_pathstring${env:COMMON_TESTRESULTSDIRECTORY} -OR- ${global:target_path}/testThe path where test results should be stored.
quietbooleanfalseA value indicating whether or not to suppress output when executing Condo.
